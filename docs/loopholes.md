# High Priority Loopholes

## LP-001 Real-Time Market Movement

Status:
Not Implemented

Problem:

Current quotes update only when the user manually refreshes.

The application does not continuously move with the market.

Current behavior:

User clicks Refresh
→ New data fetched

Desired behavior:

Market open
→ Prices move automatically

Examples:

* Price changes live
* Percentage updates live
* Watchlists update automatically
* Signals recalculate automatically

Possible Solutions:

Option A:
TanStack Query polling

* 15 sec refresh
* 30 sec refresh
* Easy implementation

Option B:
WebSockets

* True real-time
* More scalable
* Better UX

Priority:
Very High
